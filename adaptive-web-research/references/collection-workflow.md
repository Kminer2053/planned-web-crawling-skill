# Adaptive Collection Workflow

## Use This Skill For

- 조사 대상을 처음 접했을 때 구조를 빠르게 파악해야 하는 웹 수집
- HTML, JSON, PDF, 폼 POST, 단순 페이지네이션이 섞인 공공/기관 사이트 수집
- AI가 수집 전략을 먼저 세우고, 파이썬 유틸리티로 증거를 남기며 반복 확인해야 하는 작업

## Default Loop

1. `crawlkit.py probe <url>`로 첫 응답의 유형과 구조를 확인한다.
2. 응답이 HTML이면 링크, 폼, 표, JSON-LD, 페이지네이션 후보를 읽고 다음 수집 단계를 정한다.
3. 응답이 JSON이면 top-level 구조와 리스트 키를 보고 반복 요청 규칙을 정한다.
4. 응답이 PDF이면 본문 샘플을 확인하고 필요한 경우 별도 파싱 코드를 추가한다.
5. 수집 계획이 반복 요청 형태라면 `run_collection_plan.py`용 JSON 계획 파일을 만든다.
6. 결과는 항상 `--output-dir`로 저장해서, 이후 검증과 재현이 가능하게 한다.

## Planning Heuristics

- 우선순위는 공식 데이터 소스, 정적 구조, API/JSON, HTML 목록, PDF 순으로 둔다.
- 로그인, 캡차, 강한 JS 렌더링이 보이면 우회 구현보다 대안 소스 탐색을 먼저 시도한다.
- 한 번에 많은 추측을 하지 말고, probe 결과를 근거로 다음 요청을 결정한다.
- 링크 텍스트, 쿼리 파라미터, 폼 필드 이름을 그대로 증거로 남긴다.
- 중간 산출물은 삭제하지 말고 저장 경로를 답변에 남긴다.

## Local100 Reuse Notes

로컬100 프로젝트에서 재사용한 핵심 패턴:

- `urllib` + 쿠키 jar 기반 세션 유지
- GET/POST 공통 fetch 함수
- HTML에서 링크/폼/텍스트 구조 탐색
- PDF 대응 분기
- 결과 해시와 원본 스냅샷 저장

로컬100 전용 규칙은 넣지 않았다. 항목명 보정, 지역명 규칙, 수동 좌표 오버라이드 같은 도메인 로직은 제외했다.
